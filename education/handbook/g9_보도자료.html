1단계: 보도자료 — 기자가 쓸 수 있는 글
👤
박낭만

보도자료를 어떻게 써야 해? 기자한테 보내는 거잖아.

🤖
에이전트(마케팅팀)

정확해요. 기자가 그대로 쓸 수 있는 글이에요.

좋은 보도자료의 기준:

  • 기자가 수정 없이 그대로 발행할 수 있을 것
  • 기자가 추가 취재 없이 쓸 수 있을 것
  • 기자가 5분 안에 핵심을 파악할 수 있을 것
👤
박낭만

기자로서 받을 때 제일 별로였던 게 뭐야?

🤖
에이전트(마케팅팀)

세 가지예요:

  1. 첫 줄부터 "혁신적인 서비스로..."
  2. 핵심 숫자 없음
  3. 누구에게 왜 중요한지 불명확

반대로 바로 쓸 수 있었던 것:

  1. 첫 줄에 뉴스가 있음 (무슨 일이 일어났는가)
  2. 구체적 숫자·출처
  3. 이게 왜 독자에게 중요한지 명확
🎓 강사 노트 ▼ 펼치기

기자 출신의 역발상 — "받아보던 기준"이 그대로 "쓰는 기준"이 됨. 즉시 활용 가능한 경험 자산임을 강조.

2단계: 보도자료 4개 파트
👤
박낭만

보도자료 구조가 어떻게 돼?

🤖
에이전트(마케팅팀)
  • 뉴스 한 줄: 무슨 일이 일어났는가
  • 서브 헤드라인: 맥락

예:

  • 5W1H: 누가/무엇을/언제/어디서/왜/어떻게
  • 기자가 이것만 읽어도 기사 쓸 수 있게
  • 3~5문장
  • 각 소제목 = 독립적 뉴스 포인트
  • 순서: 중요한 것 먼저
  • 회사 소개 (고정 문구)
  • 문의처
3단계: 리드 작성 — 5W1H를 한 문단에
👤
박낭만

리드를 어떻게 써?

🤖
에이전트(마케팅팀)

문제:

  • 왜 중요한지 없음
  • 차별화 없음
  • 기자가 쓸 말이 없음
  • 누가: 유니콘랩
  • 무엇을: 창간호 발행
  • 왜: 제작 역량을 결과물로 증명하기 위해
  • 어떻게: 1인+AI로 직접 제작
  • 독자에게 왜 중요: 매거진 위탁사에게 포트폴리오
4단계: 헤드라인 — 뉴스가 있어야 한다
👤
박낭만

제목을 어떻게 짜?

🤖
에이전트(마케팅팀)

헤드라인 = "이게 왜 뉴스인가"를 한 줄로

  1. 최초 — "국내 최초", "처음으로"
  2. 변화 — "기존과 다른 방식", "반전"
  3. 숫자 — 구체적 수치 (1,100억, 3일, 1인)
  4. 충돌·역설 — "만들어주는 회사가 스스로 만든다"
  5. 규모 — 크거나 작거나 (극단이 뉴스)

→ "만들어 주는 회사가 자기 매거진을 직접 만들었다" = 충돌이 뉴스

뉴스가 없다. 기자가 쓸 각도가 없다.

5단계: 본문 소제목 — 각각 독립 포인트
👤
박낭만

본문 소제목은 어떻게 달아?

🤖
에이전트(마케팅팀)

소제목 하나 = 뉴스 포인트 하나예요.

기자가 기사 쓸 때 소제목 하나를 서브 앵글로 쓸 수 있어요.

  1. 각각 독립적 (앞 소제목 안 읽어도 이해 가능)
  2. 볼드 또는 ■ 기호로 구분
  3. 중요한 순서로 배치

각 소제목이 독립적인 기사 각도를 제공해요.

6단계: 보일러플레이트 — 고정 회사 소개
👤
박낭만

보일러플레이트가 뭐야?

🤖
에이전트(마케팅팀)

모든 보도자료 말미에 고정으로 들어가는 회사 소개 문단이에요.

기자가 기사 쓸 때 인용하는 "회사 소개" 부분이에요.

  1. 200자 이내 — 너무 길면 안 읽힘
  2. 사실·수치 중심 — 자랑 아닌 팩트
  3. 매번 동일 — 보도자료마다 다르면 안 됨

```
[회사명]은 [무엇을 하는 회사]다. [차별점 한 줄]. [현재 상태 또는 규모].
홈페이지: [URL]
```

7단계: 외부 배포 전 확인 사항
👤
박낭만

보도자료 쓰고 바로 뿌려도 돼?

🤖
에이전트(마케팅팀)

아니요. 외부 배포 전에 반드시 확인해야 하는 항목이 있어요.

  1. 대표/클라이언트 최종 승인 — 내용·숫자 확인
  2. 확정 필요 항목 — URL, 이메일, 날짜 빈칸 없는지
  3. 숫자·출처 — 모든 수치에 출처 있는지
  4. 배포 대상 리스트 — 어느 매체에 보낼지 확정

초안이 완성됐지만 확정 항목이 채워진 후 배포 가능하다는 뜻

기자로서 받을 때 — 날짜 빈칸, URL 오류 있는 보도자료는 신뢰도 하락했잖아요. 클라이언트 보도자료도 같은 기준이에요.
