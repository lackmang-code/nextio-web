캘린더를 어떻게 나눠야 해?

🤖
에이전트(마케팅팀)

역할: 독자 예열 + 기대감 조성

  • D-7: 내부 준비 완료 (계정 점검, 템플릿 확정)
  • D-5: 창간 예고 1 (LinkedIn — "곧 창간합니다")
  • D-4: 티저 1 (X/Threads — 제호 공개)
  • D-3: 표지 티저 (Instagram)
  • D-2: 티저 2 카운트다운
  • D-1: 창간 예고 2 (LinkedIn — "왜 이 매거진인가")

역할: 공식 발행 + 링크 공개

  • LinkedIn 창간 공지 (메인)
  • X/Threads 동시 발행 (핵심 한 줄)
  • Instagram 표지 공개

역할: 독자 후속 유입 + 수주 CTA

  • D+1: 비하인드 (어떻게 만들었나)
  • D+5, D+7: 타깃별 후킹 (학과·기업)
  • D+14: 수주 CTA
3단계: 게시 유형 — 무엇을 왜 올리는가
👤
박낭만

매일 다른 걸 올리는 거야? 뭘 언제 올려야 해?

🤖
에이전트(마케팅팀)

게시 유형이 달라요. 5개 유형이에요:

  • 무엇: "창간합니다" 공식 발표
  • 언제: D-5, D-day
  • 채널: LinkedIn 메인
  • 무엇: 기대감 유발, 아직 다 안 보여주기
  • 언제: D-4~D-2
  • 채널: X/Threads, Instagram
  • 무엇: 제작 과정, "이렇게 만들었다"
  • 언제: D+1~D+2
  • 채널: LinkedIn
  • 무엇: "왜 우리가 믿을 만한가" 증거
  • 언제: D+2~D+9
  • 채널: LinkedIn, Instagram 카드뉴스
  • 무엇: "의뢰하실 분 연락주세요"
  • 언제: D+14
  • 채널: LinkedIn
  • 예고 = 초청장 발송
  • 티저 = "특별한 게 있다" 사전 힌트
  • 비하인드 = 행사 후 포토 공유
  • RTB = 포트폴리오 배포
  • CTA = "다음 행사도 맡길게요" 유도
4단계: 원천 없이 올리지 않는다
👤
박낭만

매일 뭘 올릴지 새로 만들어야 해?

🤖
에이전트(마케팅팀)

아니요. SNS가 본문을 창작하지 않아요.

  • 창간사 원고 → LinkedIn 예고글
  • 커버스토리 요지 → X 티저
  • 표지 이미지 → Instagram
  • 제작 과정 메모 → 비하인드 글

원천이 없으면 → 빈칸으로 남겨두고 원천 먼저 완성

캘린더를 먼저 만들면서 원천 콘텐츠가 어디서 오는지 역산으로 파악할 수 있어요.
게시 D-5인데 창간사 원고가 없으면 → 창간사를 D-7 전에 완성해야 한다는 게 보임

5단계: 빈도와 채널 결정

> 발행 1주 전(D-7)에 반드시 완료해야 할 항목

왜 D-7이 중요한가?

D-7에 준비가 완료되지 않으면 D-5 예고글부터 빈칸이 된다.
이벤트 플래너의 원칙: 무대 세팅은 첫 손님 오기 전에 끝나야 한다.

SNS 게시 유형별 공식

> 5개 게시 유형 — 각 유형의 목적·구조·예시

왜 유형을 구분하는가?

같은 "게시"라도 목적이 다르면 구조가 달라야 한다.
목적 없이 올리면 → 독자가 다음 행동을 모름

5개 유형

유형목적주요 채널타이밍
① 예고/공지공식 발표, 신뢰 형성LinkedInD-5, D-day
② 티저기대감 유발, 호기심 자극X/Threads, InstagramD-4~D-2
③ 비하인드진정성 증명, 친밀감 형성LinkedInD+1~D+2
④ 근거(RTB)"왜 우리인가" 증거 제시LinkedIn, InstagramD+2~D+9
⑤ 수주 CTA의뢰·문의 유도LinkedInD+14

① 예고/공지 (Announcement)

```

  1. 핵심 한 줄 (무슨 일인지)
  2. 맥락 (왜 이걸 만들었는가)
  3. 구체적 내용 (무엇을 볼 수 있는가)
  4. CTA (어디서 볼 수 있는가)

```

```
창간합니다.

《월간 에이전트》는 AI 시대에 혼자 회사를 차리고 키우는 사람들을 위한 매거진입니다.
AI 도구를 가려주고, 실제 1인 창업자를 보여주고, 세상을 읽어줍니다.

창간호: [URL]
```

  • [ ] "무슨 일인지" 첫 줄에 있는가?
  • [ ] CTA(링크)가 포함됐는가?
  • [ ] 과장 없이 사실 중심인가?

② 티저 (Teaser)

```

  1. 힌트 (전부 말하지 않기)
  2. 숫자나 날짜 (카운트다운)
  3. 다음 게시 예고 (선택)

```

```
"D-2."

곧 공개됩니다.
혼자 회사를 차리는 사람들을 위한 매거진.

```

```
이미지: 표지 실루엣 또는 일부만 보이는 사진

캡션:
"창간까지 3일.
무엇이 나오는지는 — 아직은 비밀."
```

③ 비하인드 (Behind)

```

  1. 제작 과정 한 장면
  2. 어려웠던 점 또는 배운 것
  3. AI가 어떻게 도왔는가

```

```
창간호를 만들면서 배운 것.

혼자서 표지 디자인부터 본문 편집까지 전부 했습니다.

그 중 12개 버전의 표지를 거쳐 최종 표지가 나왔습니다.

AI와 함께라면 혼자서도 가능하다는 것을,
이 매거진 자체가 증명합니다.
```

  • [ ] 구체적 숫자나 장면이 있는가?
  • [ ] "자기 PR"이 아닌 "이렇게 했다" 중심인가?

④ 근거 (RTB — Reason to Believe)

```

  1. 주장 한 줄
  2. 근거 1~3개 (숫자·사례·경험)
  3. 독자에게 의미하는 것

```

```
콘텐츠만 주시면 됩니다.

지난 15년간 편집팀을 운영해본 경험,
AI 편집장과 30회 이상 협업한 워크플로우,
그리고 창간호 《월간 에이전트》가 그 증거입니다.

학과·연구소·기업의 브랜드 매거진 위탁을 맡습니다.
```

```
카드 1: "혼자 만든 매거진" (자기증명)
카드 2: "AI가 편집장 역할" (포트폴리오)
카드 3: "월간 발행 시스템 완비" (지속성)
```

⑤ 수주 CTA (Call to Action)

```

  1. 창간 이후 성과 또는 반응 (근거)
  2. 제공하는 것 (서비스 명확화)
  3. 연락처 또는 링크 (명확한 CTA)
  4. 마감감·긴급감 (선택)

```

```
창간 2주 — 결과를 공유합니다.

[반응 수치 또는 문의 건수]

저희는 학과·연구소·기업을 위한
온라인 매거진 위탁 발행을 합니다.

콘텐츠만 주시면 — 나머지는 저희가 합니다.

문의: [이메일/폼 URL]
```

  • [ ] 무엇을 해주는지 명확한가?
  • [ ] 연락처가 1클릭으로 가능한가?
  • [ ] 과장 없이 사실 중심인가?

상황유형
