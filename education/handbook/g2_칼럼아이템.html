1단계: 아이템 선정에 기준이 필요한 이유
👤
박낭만

아이템 선정을 그냥 감으로 하면 안 돼?

🤖
에이전트(편집제작팀)

감이 틀리진 않아요. 그런데 3가지 문제가 생겨요:

  1. 일관성 없음 — 어떤 달은 기술 중심, 어떤 달은 사회 중심 → 코너 정체성이 흔들림
  2. 코너 간 중복 — 비슷한 주제가 다른 코너에 겹쳐서 나옴
  3. 설명하기 어려움 — 고객이나 독자가 "왜 이걸 골랐어요?" 물었을 때 대답 없음

기준이 있으면 — 매달 같은 방식으로 고르고, 설명할 수 있어요.

👤
박낭만

과학지에서 편집장이 뭘 기준으로 골랐는지 몰랐는데.

🤖
에이전트(편집제작팀)

많은 편집장이 직관으로 하고, 그 직관이 오랜 경험의 축적이에요. 우리는 그걸 3가지 축으로 명시화하는 거예요.

🎓 강사 노트 ▼ 펼치기

기자 출신은 "뉴스 가치 판단"에 익숙하다. 3축이 그 기준의 명시화임을 강조.

2단계: 선정 기준 3축
👤
박낭만
🤖
에이전트(편집제작팀)

이래요:

"세상을 리딩하는 무게가 있나?"

기자로서 "이 뉴스가 중요한가?" — 똑같아요.

  • ★★★ = 패러다임 전환급
  • ★★ = 중요하지만 니치
  • ★ = 흥미롭지만 작은 변화

"우리 독자(1인 창업자)와 연결고리가 있나?"

AI 코딩 툴 = 1인 창업자 직접 활용 → 독자 연결 높음
소행성 탐사 = 아름답지만 직접 관련 없음 → 독자 연결 낮음

"다른 코너·지난 호와 겹치지 않나?"

코너가 여러 개라면 각 코너가 서로 다른 역할을 해야 해요. 같은 주제를 두 코너가 다루면 독자가 "왜 이게 두 번이야?"라고 느껴요.

3단계: 후보 풀 만들기
👤
박낭만

후보는 어떻게 만들어?

🤖
에이전트(편집제작팀)

D-14 소스 스캔 결과에서 뽑아요 (편집기획 시스템 시나리오에서 만든 루틴).

  1. 소스 레지스트리 훑어 흥미로운 것 7~10개 수집
  2. 제목·링크만 일단 나열
  3. 각각 3축 간단히 평가 (★1~3개 or 높음/중간/낮음)
  4. 상위 3~5개 추려서 최종 선정 단계로
  • 제한 없이 일단 모으기 — 좁히는 건 다음 단계
  • 중복 걱정 없이 — D-12에 코너별 중복 체크
4단계: 코너 간 역할 분리
👤
박낭만

코너가 여러 개면 어떻게 역할을 나눠?

🤖
에이전트(편집제작팀)

각 코너에 "이 코너는 이런 것만"이라는 역할 규칙이 있어야 해요.

코너역할커버 대상
① 기술 인물사람 중심누가 이 기술을 만들었나
② 편집장 칼럼기술 트렌드AI 이외 기술, 큰 그림
③ 도구 비교도구·제품AI 모델·앱 비교

→ 이 구조면 같은 "AI 코딩 툴" 주제도:

  • ③ 도구 비교 = 어떤 도구가 좋은가 (실용)
  • ② 편집장 칼럼 = 코딩이 어떻게 바뀌고 있나 (트렌드)
  • 화면 안: AI/소프트웨어
  • 화면 밖: 로봇/물리세계
  • 화면 너머: 에너지/바이오/우주
5단계: 형식 선택 — 3선 vs 1주제 심층
👤
박낭만

칼럼을 어떻게 쓸 거야? 하나를 깊게 파야 해?

🤖
에이전트(편집제작팀)

두 가지 형식 중 선택해요:

"이 달의 주목 기술 3가지" — 각 하나씩 압축

장점: 넓은 시야 / 독자가 다양하게 접함
단점: 각각 깊이가 얕음

"이달의 핵심 기술 하나 깊게"

장점: 깊이 있음 / 전문성 높음
단점: 한 달에 하나만 / 독자 관심이 분산

  • 창간호·이슈가 많은 달 → A (3선)
  • 핵심 이슈 하나가 압도적인 달 → B (심층)
  • 독자층이 일반 대중 → A / 전문가 → B
👤
박낭만

과학지에서 커버스토리는 1주제 심층이었고, 짧은 코너는 3선이었는데.

🤖
에이전트(편집제작팀)

맞아요. 콘텐츠 아키텍처가 같아요. 칼럼 분량에 따라 선택하면 돼요.

6단계: 균형 편집 — 미담과 경계편
👤
박낭만

매달 긍정 기사만 쓰면 안 돼?

🤖
에이전트(편집제작팀)

안 좋아요. "AI 만능론 선전"이 될 수 있어요.

균형 편집 원칙:

  • 미담·성공 사례 2~3호
  • 경계편(비판·위험 경고) 1호
  • "AI 데이터센터의 에너지 그림자"
  • "AI 과대광고의 거품 경고"
  • "개인정보 위험"

경계편이 있으면:

  • 콘텐츠의 신뢰도 올라감 (비판도 할 줄 안다)
  • 독자가 맹목적 추종이 아닌 판단력 생김

칼럼 아이템 3축 평가법

> 매달 칼럼 주제를 일관되게 선정하는 기준

왜 기준이 필요한가

감으로 고르면:

  • 코너 정체성이 흔들림 (달마다 달라짐)
  • 코너 간 주제 중복 발생
  • "왜 이걸 골랐어요?" 설명이 어려움
  • 매달 같은 방식으로 일관성 유지
  • 코너 역할 명확화
  • 독자·고객에게 설명 가능

축 1: 임팩트

등급기준예시
★★★패러다임 전환급AI가 일하는 방식을 바꾸는 에이전트 AI
★★중요하지만 니치특정 산업에만 영향이 큰 기술
★흥미롭지만 작은 변화편의 기능 개선 수준
  • 5년 후에도 언급될 기술인가?
  • 산업 구조를 바꾸는가?
  • 주요 미디어가 앞다퉈 보도하는가?

축 2: 독자 연결

등급기준예시
높음독자가 바로 활용 가능AI 코딩 툴, 뉴스레터 도구
중간간접 영향로봇 산업 트렌드
낮음흥미롭지만 무관소행성 탐사, 핵융합 연구
  • 독자가 이 기술을 오늘부터 쓸 수 있나?
  • 독자의 사업에 영향을 미치는가?
  • 독자가 "나도 관련 있다"고 느끼는가?

축 3: 차별화

상태기준
높음다른 코너에서 다루지 않는 영역
중간겹치지만 앵글이 다름
낮음이미 다른 코너에서 동일하게 다룸
  1. 다른 코너 담당 주제 목록 확인
  2. 지난 3개월 발행호 주제 목록 확인
  3. 겹치는 게 있으면 앵글을 바꾸거나 다른 주제 선택

3축 종합 평가표

후보임팩트독자연결차별화총점채택
★★★/★★/★높/중/낮높/중/낮
  • 임팩트 ★★★ + 독자연결 높음 + 차별화 높음 → 최우선
  • 임팩트 ★★★ + 나머지 중간 이상 → 가능
  • 임팩트 ★★ 이하 + 차별화 낮음 → 보류

코너 역할 분리 원칙

> 코너 간 중복을 방지하고 각 코너의 정체성을 유지하는 방법

역할 분리가 필요한 이유

같은 주제를 두 코너가 다루면:

  • 독자: "왜 이게 두 번이야?" — 혼란
  • 콘텐츠: 중복 작업, 품질 분산
  • 코너 정체성: 각각이 뭘 하는 코너인지 불분명

코너 역할 분리 방법

1단계: 각 코너의 "이것만" 규칙 설정
코너담당 영역담당하지 않는 것
① 인물 스토리사람·스토리기술 그 자체
② 편집장 칼럼기술 트렌드·큰 그림AI 모델 세부 비교
③ 도구 비교도구·제품·실용기술 개념·배경
2단계: 같은 주제도 앵글로 구분

"AI 코딩 툴" 주제 예시:

  • ② 편집장 칼럼: "코딩이 어떻게 바뀌고 있나" (트렌드 앵글)
  • ③ 도구 비교: "어떤 툴이 제일 좋나" (실용 앵글)

→ 같은 소재, 다른 앵글 → 중복 아님

"화면 안→밖→너머" 구성

시야를 점차 넓히는 구성 원칙:

층영역예시
화면 안AI/소프트웨어에이전트 AI, 코딩 툴, LLM
화면 밖로봇/물리세계휴머노이드 로봇, 자율주행
화면 너머에너지/바이오/우주소듐이온 배터리, 유전자편집

"AI 전문 매거진이 AI만 다루지 않는다" → 신뢰도·격 상승

코너 역할 분리 체크리스트

확인 항목완료
각 코너의 "이것만" 규칙이 문서화됐는가☐
코너별 담당 소스 레지스트리가 분리됐는가☐
아이템 선정 시 다른 코너 주제와 대조했는가☐
지난 3호 발행 주제와 중복 확인했는가☐
