1단계: 세리프가 뭔데?
👤
박낭만

에이전트야, 폰트 골라야 하는데 친구가 세리프랑 산세리프를 섞어써라고 하는데 그게 뭐야?

🤖
에이전트(총무팀)

쉽게 설명할게요. 글자에 "발"이 달려 있으면 세리프, 없으면 산세리프예요.

→ 예: 조선일보 본문체, 바탕체, Noto Serif

→ 예: 고딕체, Pretendard, 나눔고딕

"섞어써"라는 말은 — 제목엔 세리프, 본문이나 UI엔 산세리프를 쓰면 대비가 생겨서 더 세련되어 보인다는 뜻이에요.

👤
박낭만

아 그렇구나. 그럼 어떤 걸 써야 해?

🤖
에이전트(총무팀)

박낭만 씨 회사 특성을 먼저 생각해봐야 해요. 교육 교재 + 뉴스레터를 만드는 회사잖아요. 두 용도의 특성이 달라요:

그래서 이 경우 이상적인 조합은:

  • 제목·강조: 임팩트 있는 폰트 (검은고딕 같은 굵은 것)
  • 교재 본문: 명조 계열 (Noto Serif KR)
  • 화면·UI·캡션: Pretendard (산세리프)

세 가지 폰트 역할 분담이에요. 실제로 보여드릴게요.

🎓 강사 노트 ▼ 펼치기

폰트를 "예쁜 것"이 아니라 "용도에 맞는 것"으로 선택하는 관점을 에이전트가 먼저 제시한다. 비전문가가 가장 많이 실수하는 지점이 "그냥 예쁜 폰트"를 쓰는 것이다.

2단계: 폰트 시안 비교
🤖
에이전트(총무팀)

두 가지 페어링 조합을 만들어드릴게요. 실제로 교재 한 페이지처럼 보여드리면 느낌이 바로 옵니다.

  • 대제목: 검은고딕 (Black Han Sans) — 강한 임팩트
  • 소제목·기사 제목: 고운바탕 (Gowun Batang) — 격조 있는 명조
  • 본문: Noto Serif KR — 긴 글 읽기 최적화
  • UI·캡션: Pretendard — 작은 글씨 가독성
  • 제목·소제목: Pretendard Bold — 깔끔하고 현대적
  • 본문: Pretendard Regular — 통일감
  • 강조: IBM Plex Sans KR — 살짝 다른 느낌

사용 원칙

  • 인쇄 교재 본문은 반드시 Noto Serif KR (명조) — 긴 글 가독성
  • 화면 UI·버튼·태그는 Pretendard (산세리프) — 작은 크기 가독성
  • 검은고딕은 표지·대형 배너에만 — 본문에 쓰면 과해짐
  • 영문 Fraunces는 영문 전용 — 한글 입력 시 글자 깨짐

OFL 라이선스란?

SIL Open Font License. 오픈소스 폰트 표준 라이선스.

  • 개인·상업 프로젝트에 자유롭게 사용
  • 고객 납품물(PDF·인쇄물·앱)에 폰트 포함
  • 웹폰트로 서버에 올려서 서비스
  • 폰트 수정 후 재배포 (단, 같은 OFL 라이선스 유지)
  • 폰트 파일 자체를 유료로 판매

확인 방법

새 폰트 도입 시 반드시 확인:

  1. 폰트 공식 페이지에서 라이선스 파일(LICENSE.txt 또는 OFL.txt) 확인
  2. "SIL Open Font License" 또는 "OFL" 문구 있으면 안전
  3. 불확실하면 사용 중단하고 담당자에게 문의

주의

  • 마이크로소프트 오피스 번들 폰트(맑은 고딕, 굴림 등)는 MS 제품 내에서만 사용 허용 — 납품 금지
  • 네이버 나눔체: 인쇄·납품 허용, 웹폰트도 가능 (OFL 유사 라이선스)
  • 어도비 폰트: 구독 중일 때만 사용 가능 — 구독 해지 시 납품물에서도 제거해야 함
🎓 강사 노트 ▼ 펼치기

텍스트 설명보다 시각적 샘플을 바로 보여주는 것이 핵심이다. 비전문가는 폰트 이름을 들어도 모른다. 실제 글이 어떻게 보이는지를 보여줘야 선택할 수 있다.

3단계: 선택과 질문
👤
박낭만

(샘플 보고) A가 훨씬 고급스럽긴 한데... 폰트가 4가지면 너무 복잡하지 않아? 나 혼자 관리해야 하는데.

🤖
에이전트(총무팀)

맞는 걱정이에요. 근데 실제로 쓰다 보면 자연스럽게 나눠져요. 제가 규칙을 정해드리면 매번 고민 안 해도 됩니다.

  • "표지나 배너에 큰 글씨" → 검은고딕
  • "기사·교재 제목" → 고운바탕 Bold
  • "읽는 본문" → Noto Serif KR
  • "그 외 전부" → Pretendard

이 4가지만 기억하면 돼요. 헷갈리면 Pretendard로 쓰세요. 실수가 없거든요. Pretendard는 어디 써도 보기 좋아요.

👤
박낭만

아 그렇게 보니까 쉽네. 근데 이 폰트들 유료야? 회사 자료에 써도 되는 거야?

🤖
에이전트(총무팀)

좋은 질문이에요. 이거 진짜 중요한 부분이에요. 폰트 저작권 모르고 쓰다가 나중에 문제 생기는 경우 있거든요.

제가 추천드린 폰트들 전부 OFL(SIL Open Font License) 라이선스예요. 이 라이선스는:

  • 무료로 쓸 수 있음 ✅
  • 상업적으로 써도 됨 ✅
  • 인쇄물·PDF에 포함시켜도 됨 ✅
  • 고객에게 납품해도 됨 ✅
  • 유일한 금지: 폰트 자체를 유료로 파는 것 ❌

그래서 교재 인쇄, 뉴스레터 발행, 고객사에 납품하는 모든 상황에서 안전해요.

🎓 강사 노트 ▼ 펼치기

폰트 라이선스는 창업 초기에 반드시 확인해야 한다. 에이전트가 먼저 이 리스크를 짚어준다. "알아서 무료 폰트 쓰면 되지"라고 생각했다가 나중에 손해배상 청구를 받는 경우가 실제로 있다.

4단계: 설치와 관리
👤
박낭만

설치는 어떻게 해?

🤖
에이전트(총무팀)

구글 폰트(fonts.google.com)에서 전부 무료로 받을 수 있어요. 검색하면 바로 나와요.

윈도우 설치 방법:

  1. 다운로드한 .ttf 또는 .otf 파일 더블클릭
  2. "설치" 버튼 클릭
  3. 끝

설치하면 파워포인트, 한글, 어도비 같은 프로그램 다 자동으로 인식해요.

HTML·CSS에서 쓸 때는 설치 없이 구글 폰트 링크만 붙여넣으면 돼요. 제가 나중에 홈페이지 만들 때 거기서 해드릴게요.

👤
박낭만

폰트 몇 개씩 받아야 해? 굵기가 여러 개 있던데.

🤖
에이전트(총무팀)

필수만 받아도 충분해요. 이것만 받으세요:

폰트필수 굵기
검은고딕Regular (이것만 있음)
고운바탕Regular, Bold
Noto Serif KRRegular, Medium
PretendardRegular, Medium, SemiBold, Bold

Pretendard만 4가지인 이유 — 캡션·본문·소제목·강조에 각각 다른 굵기를 써야 하거든요. 나머지는 2개면 충분해요.

5단계: 실전 적용 규칙
👤
박낭만

그럼 내가 실제로 교재 만들 때 어떻게 적용해?

🤖
에이전트(총무팀)

이렇게 하면 돼요. 헷갈릴 때마다 꺼내보는 규칙표 하나 만들어드릴게요.

  • 표지 대제목: 검은고딕 / 42pt / 딥 인디고
  • 챕터 제목: 고운바탕 Bold / 32pt / 딥 인디고
  • 소제목: 고운바탕 Bold / 22pt / 딥 인디고
  • 본문: Noto Serif KR / 12pt / 딥 네이비
  • 캡션·주석: Pretendard Regular / 10pt / 그레이
  • 섹션 제목: 고운바탕 / 24px / 딥 인디고
  • 본문: Pretendard / 16px / 딥 네이비
  • 태그·버튼: Pretendard / 13px

인쇄물(pt)과 화면(px)에서 단위가 다른 거 주의하세요.

🎓 강사 노트 ▼ 펼치기

단위 차이(pt vs px)는 비전문가가 처음 만나면 헷갈리는 개념이다. 에이전트가 먼저 이 차이를 설명하면서 실수를 예방한다.

6단계: 라이선스 체크리스트
👤
박낭만

라이선스 관련된 거 문서로 정리해줄 수 있어? 나중에 직원 생기면 공유해야 할 것 같아서.

🤖
에이전트(총무팀)

좋은 생각이에요. 직원이 생기면 이런 거 따로 설명하기 번거로운데, 문서 하나 있으면 바로 공유할 수 있죠.
