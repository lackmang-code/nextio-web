1단계: 로고 "시스템"이 뭐야?
👤
박낭만

에이전트야, 로고 시스템이 뭐야? 그냥 로고 하나면 되는 거 아냐?

🤖
에이전트(총무팀)

딱 한 가지 질문으로 설명해드릴게요.

유튜브 채널 썸네일 만들 때 — 어두운 배경에 밝은 텍스트, 밝은 배경에 어두운 텍스트를 따로 써봤죠?

그것처럼 로고도 배경에 따라 다른 버전을 써야 해요.

예를 들어:

  • 흰 명함에 컬러 로고 → 잘 보임
  • 검정 가방에 같은 컬러 로고 → 안 보임 (배경에 묻힘)
  • 신문 흑백 광고에 컬러 로고 → 인쇄 불가

그래서 로고는 하나가 아니에요. 배경·상황·매체에 따라 쓸 수 있는 여러 버전이 한 세트로 준비돼야 해요. 이게 로고 시스템이에요.

유튜브 채널 했던 감각으로 이미 직관적으로 알고 계신 거예요.

👤
박낭만

아 그래서 파일이 10개가 넘어왔구나. 근데 어떤 걸 어디서 써야 해?

🤖
에이전트(총무팀)

기준을 알면 쉬워요. 로고 파일을 분류하는 두 가지 축이 있어요:

  • 컬러: 밝은 배경에 사용
  • 반전(화이트): 어두운 배경에 사용
  • 흑백: 1도 인쇄(신문·영수증·팩스)
  • 세로형(메인): 가장 기본
  • 가로형: 헤더·명함처럼 가로로 긴 자리
  • 심볼 단독: 작은 자리(앱 아이콘·SNS 프로필·QR 중앙)

이 두 축을 교차하면 — 로고 파일이 왜 여러 개인지 납득돼요.

🎓 강사 노트 ▼ 펼치기

로고 시스템을 두 가지 축(색상/형태)으로 설명하면 비전문가가 즉시 이해한다. 유튜브 썸네일 경험을 레퍼런스로 쓴 것이 포인트. 에이전트는 항상 "이미 아는 것"에서 출발해야 한다.

2단계: 파일 형식 — PNG vs SVG
👤
박낭만

근데 PNG랑 SVG 파일이 둘 다 있는데 뭐가 달라?

🤖
에이전트(총무팀)

이게 제일 중요한 거예요. 하나라도 헷갈리면 나중에 큰일 나요.

  • 확대하면 깨짐
  • 화면에 넣을 때 쓰기 편함 (포토샵, 파워포인트, SNS 업로드)
  • 배경 투명 처리 가능
  • 확대해도 깨지지 않음 (무한 확대 가능)
  • 대형 인쇄·간판·현수막에 필수
  • 웹에서도 선명하게 표시됨

예시:

  • 명함 인쇄 (3cm 크기) → PNG 300dpi OK
  • 현수막 (2m 크기) → SVG 필수 (PNG는 확대하면 깨짐)
  • 유튜브 채널 아트 (2560×1440px) → PNG
  • 웹사이트 헤더 → SVG 권장

SVG가 있으면 PNG는 언제든 만들 수 있지만, PNG에서 SVG는 만들 수 없어요.

👤
박낭만

그거 몰랐어. SVG를 꼭 받아야 하는구나.

🤖
에이전트(총무팀)

맞아요. 일반 디자이너는 PNG만 주는 경우가 있어요. "SVG도 달라"고 명시해서 요청해야 해요.

나중에 회사가 커지거나, 간판 달거나, 유튜브 채널아트 크게 쓸 때 SVG 없으면 돈 주고 다시 만들어야 해요.

🎓 강사 노트 ▼ 펼치기

SVG/PNG 차이는 디지털 환경에 친숙한 사람에게도 자주 혼동되는 개념이다. 유튜브 채널아트처럼 본인이 실제로 만들어본 것을 예시로 들면 바로 이해한다.

3단계: 반전 버전 언제 써?
👤
박낭만

반전 버전은 어두운 배경에 쓰는 거라고 했는데, 그게 언제야?

🤖
에이전트(총무팀)

뷰티 콘텐츠 에이전시라면 자주 만날 상황:

  1. 인스타 릴스 인트로: 검정 배경에 로고 → 반전 버전
  2. 유튜브 영상 오프닝: 어두운 배경 → 반전 버전
  3. 다크 모드 웹사이트: 배경이 어두울 때 → 반전 버전
  4. 고급 느낌 프레젠테이션: 슬라이드 배경 어두울 때

반전 버전이 없으면 어두운 배경에 컬러 로고 올리면 — 로고가 배경에 묻혀버리거나 보기에 어색해요.

유튜브 경험으로 치면: 밝은 썸네일 텍스트를 어두운 배경에 올리는 거랑 같은 원리예요.

👤
박낭만

아 그렇구나. 나 썸네일 만들 때 배경에 따라 텍스트 색 바꾼 게 바로 그거였구나.

🤖
에이전트(총무팀)

정확히요. 썸네일 경험 있으면 이미 직관적으로 알고 있는 거예요. 로고 시스템은 그 원리를 공식화한 거예요.

4단계: 사용 금지 규칙
👤
박낭만

로고 쓸 때 하면 안 되는 게 있어?

🤖
에이전트(총무팀)

있어요. 그리고 이게 생각보다 중요해요.

가장 흔한 실수:

→ 로고가 찌그러져 보이고, 브랜드 신뢰도 하락

→ 브랜드 아이덴티티 훼손. 어울리지 않으면 흑백 버전을 써야 해

→ 안 보이거나 지저분해 보임. 반전 버전 또는 흰 박스 배경 사용

→ 글자 다 뭉개짐. 이 자리엔 심볼(단독 아이콘) 써야 함

→ 원본 로고가 훼손됨

👤
박낭만

아 4번이 나야. 앱 아이콘에 로고 다 넣으려고 했는데.

🤖
에이전트(총무팀)

자주 하는 실수예요. 앱 아이콘·SNS 프로필 사진 같은 작은 정사각 자리에는 심볼(단독 아이콘)만 써야 해요. 메인 로고는 텍스트가 포함되어 있어서 작게 쓰면 글자가 뭉개져요.

유니콘랩 예시로 보면:

  • 인스타 프로필 사진 → 심볼(U 아이콘) 만 쓰는 버전
  • 유튜브 채널 아트 → 가로형 메인 로고

자리에 맞는 버전이 있어요.

로고 사용 규칙

> DO / DON'T 원칙 — 모든 구성원·파트너사 공유용

✅ DO — 이렇게 쓰세요

#규칙설명
1배경에 맞는 버전 선택밝은 배경 → 컬러 / 어두운 배경 → 반전 / 흑백 인쇄 → 흑백
2작은 자리엔 심볼 사용SNS 프로필·앱 아이콘 → 심볼(단독) / 메인 로고는 최소 120px 이상
3원본 비율 유지크기를 조절할 때는 반드시 비율 고정(Shift+드래그)
4여백(보호 영역) 확보로고 주변 'U' 높이의 절반 이상 빈 공간 유지
5SVG를 인쇄에 사용현수막·간판·포스터 등 50cm 이상 → SVG 파일 필수

❌ DON'T — 절대 하지 마세요

#금지 사항이유
1비율 왜곡 (가로/세로 따로 조절)로고가 찌그러져 브랜드 신뢰도 하락
2임의 색상 변경브랜드 아이덴티티 훼손 — 안 어울리면 흑백 버전 사용
3복잡한 배경에 컬러 로고 직접 올리기로고가 배경에 묻힘 — 반전 버전 또는 흰 박스 배경 사용
4그림자·외곽선·뱃지 효과 추가원본 로고 디자인 훼손
5메인 로고를 작은 자리(120px 미만)에 사용텍스트 뭉개짐 — 심볼 버전 사용
6PNG를 대형 인쇄에 사용픽셀 깨짐 — SVG 필수
7허락 없이 타인에게 제공상표권 침해 가능성

여백(보호 영역) 규칙

```
[최소 여백]
↕ U높이 ÷ 2
[여백]← [로고] →[여백]
↕ U높이 ÷ 2
[최소 여백]
```

이 영역 안에 다른 텍스트·이미지·다른 로고가 침범하면 안 됩니다.

로고 사용 허락 정책

  • 파트너사·협력사가 우리 로고를 사용할 때 → 사전 서면 허락 필요
  • 이 규칙 문서를 파트너사에 함께 공유할 것
  • 무단 사용 발견 시 → 즉시 수정 요청
5단계: 여백(보호 영역)
👤
박낭만

여백이 필요하다고도 하던데, 그게 뭐야?

🤖
에이전트(총무팀)

로고 주변에 아무것도 없는 공간을 강제로 유지하는 거예요.

왜 필요하냐: 로고 바로 옆에 다른 텍스트나 이미지가 붙으면 로고가 독립적으로 보이지 않아요. 눈이 로고를 인식하기 전에 다른 것들이 방해해요.

규칙: 로고의 "U" 높이의 절반 이상을 로고 주변 여백으로 확보

예를 들어 U가 20px 높이면 → 주변 10px는 빈 공간이어야 해요.

이 여백 안에 다른 요소(텍스트, 다른 로고, 사진)가 침범하면 안 돼요.

유튜브 썸네일로 치면: 텍스트 너무 가장자리 붙이면 유튜브가 짜르는 것처럼, 로고도 공간이 필요해요.

👤
박낭만

실제로 어떻게 확인해?

🤖
에이전트(총무팀)

디자인 소프트웨어(Figma, Adobe XD, Photoshop)에서 "여백 가이드" 레이어를 만들어두거나, 로고 주변에 투명 박스를 두면 돼요.

실용적인 방법: 로고를 사용할 때 그 주변 공간에 "다른 것 안 넣는다" 원칙만 지켜도 충분해요. 로고 옆에 텍스트 바짝 붙이지 않기.

🎓 강사 노트 ▼ 펼치기

여백(보호 영역) 개념은 브랜드 디자인의 기초다. 이것을 지키는 것과 무시하는 것의 차이가 전문 브랜드와 아마추어 브랜드의 첫 번째 차이다.

6단계: 파일 관리 방법
👤
박낭만

이 파일들을 어떻게 관리해야 해? 여기저기 산재하면 나중에 못 찾겠다.

🤖
에이전트(총무팀)

폴더 구조 만드는 게 제일 중요해요.

권장 폴더 구조:

```
📁 브랜드에셋/
📁 로고/
📁 PNG/ (화면·문서용)
- 로고메인컬러.png
- 로고메인반전.png
- 로고메인흑백.png
- 로고가로형컬러.png
- 심볼_컬러.png
📁 SVG/ (인쇄·벡터용)
- 로고메인컬러.svg
- 로고메인반전.svg
- 로고메인흑백.svg
- 로고가로형컬러.svg
- 심볼_컬러.svg
- 로고_가이드.pdf (사용 규칙 요약)
```

규칙:

  • 파일명에 버전 번호 넣지 않기 (로고v2final_진짜최종.png 같은 거 금지)
  • 원본은 SVG → 폴더에서 수정 금지 (편집할 일 있으면 복사해서 편집)
  • 디자이너한테 다음 번에 수정 요청할 때 이 폴더 기준으로 말하기
👤

저 지금 폴더 정리 안 되어 있는데, 오늘 정리해야겠다.

에이전트(총무팀)

박낭만

디자이너한테 로고 의뢰할 때 어떻게 해야 해? 뭘 요청해야 돼?

🤖
에이전트(총무팀)

디자이너한테 "로고 만들어주세요"만 하면 안 돼요. 구체적으로 요청해야 해요.

납품 요청서에 넣을 것:

  • SVG 원본 (필수)
  • PNG 투명 배경 각 버전 (300dpi 이상)
  • 컬러 / 반전 / 흑백 각각
  • 세로형(메인) / 가로형 / 심볼 단독
  • 인쇄용 (300dpi 이상)
  • 웹용 (72dpi)
  • 앱 아이콘 (512×512px, 1024×1024px)
  • 브랜드 가이드 간략 PDF (색상 코드, 여백 규칙 포함)
  • 폰트 이름 알려달라고

이걸 계약서나 발주서에 명시해야 "이거 없는데요?"라고 나중에 추가 요금 청구하는 상황 방지돼요.

로고 디자이너 납품 체크리스트

> 로고 의뢰 계약 시 이 목록을 발주서에 첨부하거나 구두로 명시 요청할 것

A. 파일 형식

#항목완료
A1SVG 벡터 원본 (전 버전)☐
A2PNG 투명 배경, 300dpi 이상 (전 버전)☐
A3웹용 PNG 72dpi (전 버전)☐
A4앱 아이콘용 PNG: 512×512px + 1024×1024px☐

B. 버전 (총 6종 이상)

#항목완료
B1메인 로고 컬러 (밝은 배경용)☐
B2메인 로고 반전 (어두운 배경용, 화이트)☐
B3메인 로고 흑백 (1도 인쇄용)☐
B4가로형 로고 컬러☐
B5심볼(아이콘) 단독 — 정사각 자리용☐
B6앱·QR 전용 심볼☐

C. 브랜드 정보 (디자이너에게 받아야 할 것)

#항목완료
C1브랜드 컬러 HEX 코드 전체 (주 색상·보조 색상)☐
C2워드마크(로고 텍스트)에 사용한 폰트 이름☐
C3여백(보호 영역) 규칙 치수☐
C4최소 크기 기준 (인쇄mm / 화면px)☐
C5브랜드 가이드 간략 PDF (가능하면)☐

D. 추가 확인

#항목완료
D1모든 파일에 투명 배경 적용됐는지 확인☐
D2SVG 파일에서 텍스트가 아웃라인(패스) 변환됐는지 확인☐
D3파일명 규칙 통일됐는지 확인 (버전 표기 v2final 없애기)☐
D4납품 받은 파일 열어서 실제로 작동하는지 확인☐

주의사항

> "SVG 원본 못 드려요"라고 하는 디자이너 = 원본 파일 갖고 나중에 추가 비용 청구 가능성
> → SVG 납품을 계약서에 명시할 것

> 텍스트 아웃라인 미변환 SVG = 열면 폰트 깨짐 (폰트가 설치 안 된 컴퓨터에서)
> → "폰트 아웃라인 변환해달라"고 명시 요청

납품 후 즉시 할 일

  1. 위 체크리스트 전부 확인
  2. 폴더 구조에 맞게 정리 (PNG/, SVG/ 분리)
  3. 로고_가이드.md 작성 (이 회사에 맞는 사용 규칙)
  4. 팀원·파트너사에 가이드 공유

로고를 다른 사람한테 사용하게 해도 돼?

🤖
에이전트(총무팀)

허락 없이 다른 사람이 쓰면 안 돼요. 로고는 상표권(또는 저작권)이 있어요.

관련해서 알아야 할 것:

  • 파트너사 자료에 들어가는 경우 → 사전 서면 허락 필요
  • 고객이 제안서에 "파트너: OO" 하고 우리 로고 넣을 때 → 미리 허락받게 하기
  • "협력사 로고" 섹션에 넣을 때 → 상대방 허락 필요
  • 모르고 쓰면 나중에 분쟁

보통 파트너십 계약서에 "로고 사용 허락" 조항을 넣어요.

👤
박낭만

그런 게 계약서에 있어야 하는구나.

🤖
에이전트(총무팀)

맞아요. 그리고 로고 사용 가이드 문서를 파트너사에 미리 공유하면 더 좋아요. "이렇게만 써주세요"가 명확하면 나중에 잘못 쓰는 경우도 줄어요.

🎓 강사 노트 ▼ 펼치기

로고의 법적 보호(상표권·저작권)와 사용 허락 개념은 1인 창업자가 자주 놓치는 부분이다. 에이전트가 미리 알려주는 것이 "선배" 역할이다.
