1단계: 홈페이지가 꼭 필요해?
👤
박낭만

에이전트야, 홈페이지가 꼭 있어야 해? 뉴스레터는 이메일로 하는 건데.

🤖
에이전트(총무팀)

홈페이지 없이 시작하는 사람도 있어요. 하지만 빠르게 필요해지는 상황이 와요.

세 가지 상황:

  1. 약사·의사가 "이게 어떤 뉴스레터예요?"라고 물을 때 → 링크 하나 보내면 끝
  2. 구독 신청 폼 → 이메일로만 받기엔 한계 (홈페이지에 폼 달면 자동화 가능)
  3. 광고주나 협력사 미팅 → "홈페이지 보여주세요"가 반드시 나옴

뉴스레터만 있으면 되는 것 같지만 — 홈페이지는 뉴스레터의 "본부" 역할을 해요. 구독 신청 받고, 서비스 설명하고, 신뢰를 쌓는 곳.

박낭만 씨 MR 경험으로 치면: 제약회사 홈페이지가 없으면 병원에서 신뢰하기 어려운 것처럼, 뉴스레터도 "공식 주소"가 있어야 해요.

👤
박낭만

그렇게 생각하니까 당연히 있어야 하네. 그러면 어떻게 만들어?

🤖
에이전트(총무팀)

방법이 4가지 있어요. 비용·난이도·시간이 다 달라요:

방법비용난이도시간
노션 페이지무료매우 쉬움1~2시간
워드프레스월 1~5만원중간1~2일
스퀘어스페이스/Wix월 2~5만원쉬움반나절
HTML 직접 제작도메인비만 (1~2만원/년)어려움1~3일

직접 코딩할 필요 없이 — AI한테 HTML 코드 작성 요청하고 GitHub Pages에 올리면 도메인비만으로 운영 가능한 방법도 있어요. 이게 요즘 1인 창업자가 가장 많이 쓰는 방법이에요.

👤
박낭만

GitHub Pages? 그게 뭐야?

🤖
에이전트(총무팀)

GitHub는 코드를 보관하는 창고 같은 곳인데 — 홈페이지 파일을 그곳에 올리면 무료로 인터넷에 공개해줘요.

즉: 내 홈페이지 파일 → GitHub에 올리기 → 인터넷에서 접속 가능

비용: 도메인(1~2만원/년)만 내면 됨. 호스팅 무료.

유니콘랩 홈페이지도 이 방식으로 만들어졌어요. AI가 HTML 코드를 짜고, 그걸 GitHub Pages에 올린 거예요.

코딩을 전혀 몰라도 — "이런 홈페이지 만들어줘"라고 AI한테 말하면 코드가 나와요. 그걸 GitHub에 올리면 돼요.

🎓 강사 노트 ▼ 펼치기

홈페이지 제작에 코딩이 필요하지 않다는 것, AI가 코드를 쓴다는 것이 이 교육의 핵심 메시지다. 유니콘랩 자체가 이 방식의 실제 사례임을 명시한다.

2단계: 홈페이지에 뭐가 들어가야 해?
👤
박낭만

홈페이지에 뭘 넣어야 해? 그냥 소개 글만 있으면 돼?

🤖
에이전트(총무팀)

소개글만 있으면 방문자가 "근데 어떻게 구독해요?"를 못 찾아요.

필수 6개 섹션:

→ "한 줄로 무엇을 하는 회사인지" + 구독 버튼

→ "왜 이 뉴스레터인가" 스토리 (2~3문단)

→ "매호 어떤 내용이 나오나" 설명

→ "이런 내용이에요" 예시 1~2개 (독자 신뢰 급상승)

→ 이메일 입력 폼 + 버튼 (가장 중요한 CTA)

→ 이메일 주소·SNS 링크

이 6개가 있으면 방문자가 홈페이지에서 이탈하지 않고 구독까지 이어져요.

👤
박낭만

사례가 없는 상태에서 홈페이지를 어떻게 해?

🤖
에이전트(총무팀)

없으면 샘플을 만드세요. 첫 호 일부를 공개하거나, "예시 뉴스레터 보기" 링크를 넣으면 돼요.

사람들이 구독 결정을 할 때 — "어떤 내용인지 먼저 보고 싶다"가 가장 큰 장벽이에요. 이걸 없애주는 게 샘플이에요.

창간 전에 홈페이지 론칭하는 방법:

  1. 샘플 뉴스레터 1편을 먼저 만들어서 PDF로 올리기
  2. "론칭 예정 — 사전 등록하기" 방식 (기대감 유발)
  3. 소셜미디어에서 미리 "어떤 뉴스레터냐"를 1~2주 공유 후 론칭

박낭만 씨는 MR 경험으로 의학 정보를 이미 갖고 있잖아요. 과거에 의사한테 설명했던 내용 중 하나를 뉴스레터 샘플로 만들면 돼요.

3단계: 도메인은 어떻게 정해?
👤
박낭만

도메인이 뭐야? 그것도 필요해?

🤖
에이전트(총무팀)

도메인은 홈페이지 주소예요. www.내회사이름.com 같은 것.

없어도 GitHub Pages가 주소를 줘요 (예: username.github.io). 근데 이 주소는 전문적으로 보이지 않아요.

도메인을 사면: www.medinfo-weekly.com 같은 깔끔한 주소가 생겨요. 비용은 연 1~2만원.

  1. 짧고 기억하기 쉬운 것
  2. 서비스 이름·콘셉트 반영
  3. .com 우선 (가장 신뢰도 높음), 없으면 .kr 또는 .io
  4. 발음하기 어려운 특수문자·숫자 피하기

박낭만 씨 뉴스레터 이름이 뭐예요?

👤
박낭만

"메디뷰"라고 하려고. 의학 정보 뷰 같은 느낌으로.

🤖
에이전트(총무팀)

좋아요. 도메인 후보:

  • mediview.kr
  • medi-view.com
  • mediview.io
  • getmediview.com (자주 쓰는 패턴 — "subscribe.get~")

도메인 등록 사이트 (gabia.com, hosting.kr 등)에서 검색하면 사용 가능 여부 즉시 확인돼요.

도메인 선정 기준

> 홈페이지 주소 결정 가이드

도메인이란

www.회사이름.com 처럼 인터넷에서 내 홈페이지를 찾는 주소.

비용: 연 1~2만원 (도메인 등록 사이트)

도메인 선택 5원칙

#원칙이유
1짧을수록 좋음 (15자 이내)기억하기 쉽고 입력 오류 적음
2서비스 이름 반영"이게 뭐하는 곳인지" 도메인에서 짐작 가능
3.com 우선전 세계 가장 많이 쓰이는 확장자 — 신뢰도 높음
4특수문자·숫자 피하기발음·기억이 어렵고 혼동 유발
5오타 방지비슷한 도메인이 없는지 확인

확장자 선택 가이드

확장자특징추천 상황
.com전 세계 표준, 최고 신뢰도1순위
.co.kr한국 기업 공식 느낌.com 없을 때 한국 사업
.kr짧고 한국적한국 독자 타겟
.io테크·스타트업 느낌디지털 서비스
.net.com 없을 때 차선책

도메인 등록 방법

  1. 가비아 (gabia.com) — 국내 가장 많이 씀
  2. 후이즈 (whois.co.kr)
  3. Namecheap (국제)
  4. Google Domains (구글 계정으로 간편)

도메인 vs 호스팅

용어비유비용
도메인집 주소연 1~2만원
호스팅집 건물 (서버)월 1~5만원
GitHub Pages무료 호스팅0원

도메인 연결 방법 (GitHub Pages 기준)

  1. GitHub Pages에서 내 홈페이지 주소 확인 (username.github.io)
  2. 도메인 등록 사이트에서 DNS 설정 → GitHub Pages 서버 주소 입력
  3. GitHub 저장소 Settings → Pages → Custom domain에 내 도메인 입력
  4. HTTPS 인증서 자동 발급 (15분~1시간 소요)

자세한 설정은 AI에게 "GitHub Pages에 내 도메인 연결하는 방법" 질문하면 단계별로 안내받을 수 있음.

4단계: SEO가 뭐야?
👤
박낭만

홈페이지 만들 때 SEO도 신경 써야 한다고 하던데, 그게 뭐야?

🤖
에이전트(총무팀)

SEO(Search Engine Optimization) — 구글 검색에서 내 홈페이지가 잘 나오게 하는 것.

의사나 약사가 구글에 "의학 정보 뉴스레터"를 검색했을 때 내 홈페이지가 보여야 하잖아요.

기본 SEO 설정 3가지:

```html
<title>메디뷰 | 의사·약사를 위한 의학 정보 뉴스레터</title>
```
→ 구글 검색 결과에 보이는 제목

```html
<meta name="description" content="주 1회 의사·약사를 위한 최신 임상 연구와 약품 정보를 큐레이션합니다." />
```
→ 검색 결과 제목 아래 나오는 설명 (클릭율에 영향)

```html
<html lang="ko">
```
→ 한국어 사이트임을 구글에 알림

이 세 가지만 제대로 해도 기본 SEO는 완료예요.

👤

저 코딩 모르는데 이런 거 어떻게 해?

🤖
에이전트(총무팀)

AI한테 이렇게 말하면 돼요:

그러면 SEO 설정이 자동으로 들어간 HTML 파일이 나와요.

코드를 이해하지 못해도 괜찮아요. "어떻게 동작하는지"만 알면 — AI가 코드를 쓰고, 사람은 결과를 확인하면 돼요.

🎓 강사 노트 ▼ 펼치기

"코드를 모르는 게 문제가 아니다, AI한테 말하는 방법을 알면 된다"는 것이 이 교육의 핵심이다. 박낭만이 SEO 설정 방법을 이해하면서도 직접 코딩하지 않는 방법을 배우는 것이 포인트다.

5단계: 모바일 최적화
👤
박낭만

의사·약사들이 폰으로 볼 수도 있잖아. 모바일에서도 잘 보여야 하는 거야?

🤖
에이전트(총무팀)

필수예요. 요즘은 절반 이상이 모바일로 접속해요.

모바일 최적화 기본 설정:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

이 한 줄이 있으면 — 폰 화면에 맞게 자동으로 레이아웃이 조정돼요.

그리고 디자인 원칙:

  • 폰트 크기 최소 14px (폰에서 읽을 수 있는 최소)
  • 버튼 크기 최소 44×44px (손가락으로 탭하기 편한 크기)
  • 스크롤로 읽기 편한 단일 컬럼 레이아웃

AI한테 홈페이지 만들어달라고 할 때 "모바일 최적화 포함해서"라고 하면 자동으로 적용돼요.

👤
박낭만

테스트는 어떻게 해?

🤖
에이전트(총무팀)

두 가지 방법:

  1. 실제 폰으로 보기: 홈페이지 URL을 폰에서 열어서 직접 확인
  2. Chrome 개발자 도구: PC에서 F12 → 모바일 미리보기 모드

저는 항상 두 번째 방법을 권해요. PC에서 Chrome 개발자 도구 열고 → 모바일 화면 크기로 전환하면 → 폰에서 어떻게 보이는지 바로 확인 가능해요.

6단계: CTA 버튼이 핵심
👤
박낭만

구독 버튼이 중요하다고 했는데, 어디에 어떻게 넣어야 해?

🤖
에이전트(총무팀)

CTA(Call To Action) — 방문자에게 "이걸 해주세요"를 요청하는 버튼이에요.

뉴스레터 홈페이지에서 CTA는 구독 신청 버튼 하나예요. 이것 하나에 집중해야 해요.

원칙:

  1. 첫 화면에 바로 보여야 함 — 스크롤하지 않아도
  2. 명확한 문구 — "구독하기"보다 "무료로 구독하기 →"가 클릭율 높음
  3. 눈에 띄는 색상 — 브랜드 포인트 컬러로
  4. 반복 배치 — 첫 화면 + 중간 + 맨 아래 (3번 이상)

잘못된 예: "더 알아보기" (모호함)
잘 된 예: "매주 화요일, 의학 정보 받아보기 →" (구체적 약속)

박낭만 씨 뉴스레터는 언제 발송해요?

👤
박낭만

매주 목요일에 하려고.

🤖
에이전트(총무팀)

그러면: "매주 목요일, 의사·약사 위한 의학 정보 받기 →"

이게 버튼 텍스트가 될 수 있어요. 언제, 누구를 위해, 뭘 받는지 — 세 가지가 다 담겨 있어요.

🎓 강사 노트 ▼ 펼치기

CTA 문구는 추상적이면 안 되고 구체적인 약속이어야 한다. 에이전트가 사용자의 정보(발송 요일)를 써서 즉시 사용 가능한 CTA를 만들어주는 것이 선배 역할이다.

7단계: 구독 폼 연결
👤
박낭만

구독 폼은 어떻게 만들어?

🤖
에이전트(총무팀)

직접 만들지 않아도 돼요. 뉴스레터 플랫폼에서 제공해요.

추천 플랫폼:

  • 스티비 (stibee.com) — 국내 가장 많이 씀, 한국어 지원, 구독자 1000명까지 무료
  • Mailchimp — 글로벌 표준, 500명까지 무료
  • ConvertKit — 콘텐츠 크리에이터에 특화

이 중 하나에 가입하면 → 임베드 코드를 줘요.

임베드 코드를 홈페이지에 붙여넣으면 → 구독 폼이 홈페이지에 자동으로 생겨요.

AI한테 이렇게 말하면 돼요:

👤
박낭만

그러면 구독자 관리도 스티비에서 하면 되는 거야?

🤖

홈페이지 공개 전 체크리스트

> 공개 전 이 모든 항목 확인 후 공개

A. 내용 확인

#항목완료
A1회사명·서비스 이름이 정확한가☐
A2전체 텍스트 맞춤법 검사 완료☐
A3연락처 정보(이메일·전화)가 정확한가☐
A4사업자 등록번호 정확한가☐
A5구독 취소 방법 안내가 있는가 (뉴스레터 법적 의무)☐

B. 기능 확인

#항목완료
B1모든 링크·버튼 클릭 확인 (깨진 링크 없는지)☐
B2구독 폼 테스트 (실제 이메일로 가입 완료 확인)☐
B3이미지가 모두 표시되는가 (로고 포함)☐
B4페이지 로딩 속도가 너무 느리지 않은가☐
B5HTTPS 적용됐는가 (주소창에 🔒 자물쇠 확인)☐

C. SEO 확인

#항목완료
C1브라우저 탭에 올바른 타이틀이 표시되는가☐
C2메타 설명(description)이 입력됐는가☐
C3lang="ko" 설정됐는가☐
C4OG 태그가 있는가 (카카오톡 공유 시 미리보기)☐

D. 모바일 확인

#항목완료
D1실제 폰에서 접속해서 확인☐
D2텍스트가 너무 작지 않은가 (14px 이상)☐
D3버튼이 손가락으로 탭하기 충분히 큰가☐
D4가로 스크롤이 생기지 않는가☐
D5구독 폼이 모바일에서도 작동하는가☐

E. 공개 후 즉시 할 일

#항목
E1Google Search Console에 사이트맵 제출 (구글 색인 요청)
E2소셜미디어에 홈페이지 오픈 공지
E3주변 지인 5명에게 링크 공유 → 피드백 받기
E4첫 구독자 등록 확인 (구독 폼 작동 재확인)

자주 빠뜨리는 항목

  1. 구독 취소 링크 — 스팸법상 의무. 뉴스레터 플랫폼(스티비 등)에서 자동 삽입됨.
  2. 개인정보 처리 방침 링크 — 이메일 수집 시 법적 필수. 간단하게라도 만들 것.
  3. 모바일 구독 폼 테스트 — PC에서만 확인하면 놓침.
