스타트업 창업 경험자는 IR 피칭에 익숙하다. 제안서의 "고객 중심" 원칙을 IR과의 차이로 설명하면 이해가 빠르다.

5단계: 파트 3 — 진행 방식
👤
박낭만

파트 3은 어떻게 써?

🤖
에이전트(영업팀)

고객이 "계약하면 어떻게 되는지"를 알아야 결정해요.

표준 프로세스:
```
착수 → 현황 분석 → 전략 설계 → 실행·제작 → 성과 확인 → 정기 지속
```

이걸 제안서에서는 표 형식 + 기간 + 역할 분담으로 표현:

단계기간우리 작업고객 작업
착수·현황 분석1주브랜드 분석·시장 파악기존 콘텐츠 자료 제공
전략 설계1주방향·캘린더 작성방향 컨펌
실행·제작매월콘텐츠 제작·게시피드백
성과 분석분기 1회보고서 작성—

핵심: 고객 부담이 최소처럼 보이게. "우리가 다 한다, 고객은 이것만"

박낭만

견적은 어디에 넣어?

🤖
에이전트(영업팀)

제안서에 넣는 경우와 안 넣는 경우가 있어요:

  • 미팅에서 이미 예산 범위가 확인된 경우
  • "범위 제시" 형태로 (정확한 금액 아닌 최소~최대)
  • 아직 규모 확정 전
  • "제안서 검토 후 견적 별도 안내" 문구로 대체

처음 제안서는 "견적은 별도 협의" 방식이 안전해요. 상대방이 읽다가 "비싸다"고 판단하고 닫아버릴 수 있거든요.

고객 언어 가이드

> 제안서의 모든 문장을 고객의 언어로 바꾸는 방법

고객 언어란?

고객이 내부 회의에서 하는 말 = 고객 언어

내가 쓰는 말이 아니라, 고객이 쓰는 말로 제안서를 써야 한다.

변환 원칙 5가지

원칙 1: 전문 용어 → 일상 언어
전문 용어고객 언어
콘텐츠 전략 수립뭘 올려야 할지 방향 잡기
마케팅 채널 최적화어디서 홍보해야 반응이 올지
리텐션 향상한 번 온 고객이 또 오게 만들기
브랜드 아이덴티티회사 느낌 통일
원칙 2: 추상 → 구체 (숫자 포함)
추상 표현구체 표현
효과적인 홍보3개월 만에 팔로워 500명 → 3,000명
저비용 운영월 30만 원으로 매달 발행
빠른 제작착수 후 14일 안에 창간호 발행
원칙 3: "우리" 중심 → "고객" 중심
우리 중심고객 중심
저희는 최고의 서비스를 제공합니다[기관명]의 성과가 더 잘 보이게 됩니다
전문 팀이 작업합니다담당자님이 직접 신경 쓰지 않으셔도 됩니다
탁월한 결과물을 만듭니다평가단이 알아볼 수 있는 형태로 나옵니다
원칙 4: 미팅에서 들은 말 그대로

미팅 기록을 제안서에 직접 넣는다:

> 미팅에서 들은 말: "요즘 학생 모집이 너무 힘들어요. 우리 학과 좋은 건 아는데..."

> 제안서 파트 1에: "우수한 성과가 있는데 외부에서는 잘 모르는 상황이시죠. 학과의 이야기가 밖으로 잘 나가지 않고 있는 겁니다."

원칙 5: 동사는 행동형으로
수동형행동형
콘텐츠가 제작될 예정입니다매달 8개 콘텐츠를 만들어 올립니다
지원이 이루어집니다담당자 전화 한 통이면 처리됩니다

제안서 3파트 구조

> B2B 서비스 제안서의 표준 구조 — 복사해서 바로 사용

핵심 원칙

> 제안서 = 고객의 이야기 (IR 피칭 ≠ 제안서)

IR 피칭제안서
독자투자자잠재 고객
목적투자금 받기서비스 계약
강조점성장 가능성고객의 문제 해결
핵심 질문"이 회사가 클까?""내 문제를 해결해줄까?"

제안서 분량 기준

타입분량상황
원페이저형1~2페이지첫 제안 (미팅 전)
표준형3~5페이지미팅 후 정식 제안

파트 1: 문제 제기 (Why Change?)

  • 고객의 언어로 (내부 회의 말투)
  • 통계·사례로 공감대 강화
  • 현재 방식의 한계를 구체적으로
나쁜 예좋은 예
"콘텐츠 전략 부재로 마케팅 효율성이 저하됩니다""인스타에 꾸준히 올리는데 팔로워가 안 늘죠?"
"기존 방식의 한계가 있습니다""종이 소식지, 작년에 몇 번 발행하셨나요?"

```
[현재 상황] 요즘 __는 __ 때문에 어려운 상황입니다.
[구체적 문제] 실제로 __ 하시는 분들 대부분이 __ 문제를 겪고 있어요.
[기존 방식 한계] 지금까지 __로 해결하려 했지만 __ 한계가 있습니다.
[공백] 결국 남은 건 ___ — 이 부분이 비어 있습니다.
```

파트 2: 솔루션 (What We Offer)

① 구체적인 서비스 내용
  • "도와드리겠습니다" X
  • "월 8개 콘텐츠 제작 + 마케팅 캘린더 + 분기 성과 분석" O
② 왜 나인가 (RTB 3개)

핵심 주장 1개 → 근거(RTB) 3개

```
[핵심 주장]: __만큼 __를 잘 아는 곳은 없습니다
RTB 1: [근거 — 경험]
RTB 2: [근거 — 방법론]
RTB 3: [근거 — 결과]
```

③ 증거 (자기증명)
  • "말이 아닌 결과물을 먼저 보여드립니다"
  • 포트폴리오, 실적, 샘플 링크

> "직접 의류 브랜드를 창업·운영했기 때문에 — 어디서 막히는지 알고 있습니다"
→ "현장을 안다"는 근거

1단계: 구조 확인

#항목완료
1파트 1 (문제 제기)가 있는가☐
2파트 2 (솔루션)가 있는가☐
3파트 3 (진행 방식)가 있는가☐
4CTA (다음 단계)가 있는가☐

2단계: 고객 중심 확인

#항목완료
5파트 1에서 고객의 언어를 쓰고 있는가☐
6"우리가 최고입니다"형 표현이 없는가☐
7고객이 "맞아, 바로 그거야" 할 문장이 있는가☐
8미팅에서 들은 말이 반영됐는가☐

3단계: 증거 확인

#항목완료
9파트 2에 포트폴리오·사례가 포함됐는가☐
10숫자·기간이 구체적인가 (추상 표현 없는가)☐
11RTB(Why Me 근거)가 3개 이상 있는가☐

4단계: 발송 준비 확인

#항목완료
12기관명·담당자명 오탈자 없는가☐
13PDF 형식인가 (편집 불가)☐
14파일 크기 5MB 이하인가☐
15파일명에 기관명 포함됐는가☐
16이메일 제목이 개인화됐는가☐
