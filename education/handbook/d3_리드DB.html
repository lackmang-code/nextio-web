1단계: 리드DB가 뭐야?
👤
박낭만

에이전트야, 리드DB가 뭐야? 보험할 때 고객 명단이랑 다른 거야?

🤖
에이전트(영업팀)

보험 고객 명단과 비슷한데 — 한 가지 큰 차이가 있어요.

보험 고객 명단: 이미 계약한 사람 관리
리드DB: 아직 계약 안 한 잠재 고객 관리

보험설계사 경험으로 비유하면:

  • 계약 고객 명단 = CRM (고객 관계 관리)
  • 리드DB = 미래 계약 가능 목록 (파이프라인)

차이는 — 리드DB에는 "아직 아무것도 결정 안 된 사람"도 들어가요.

👤
박낭만

그러면 뭘 기록해야 해?

🤖
에이전트(영업팀)

최소 5개 항목이에요:

  1. 기관/회사명 — 누구
  2. 유형 — 어떤 종류 (학과, 기업, 기관 등)
  3. 현재 상태 — 빈자리 있는지
  4. 등급 — A/B/C
  5. 연락처 — 컨택 단서 (이메일, 담당자명 등)

추가로 중요한 것:

  1. 컨택 날짜 — 언제 연락했는지
  2. 반응 — 응답/거절/보류

이 7가지가 기본 컬럼이에요.

🎓 강사 노트 ▼ 펼치기

보험설계사 경험자는 "명단 관리"를 이미 알고 있다. 리드DB의 차이점은 "아직 결정 안 된 사람도 관리 대상"이라는 점을 강조하는 것이 포인트다.

2단계: 핵심 컬럼 — 발행 유무
👤
박낭만

가장 중요한 컬럼이 뭐야?

🤖
에이전트(영업팀)

온라인 매거진 비즈니스라면 — "발행 유무"가 가장 중요한 컬럼이에요.

왜냐면: 이미 매거진 발행 중인 곳에는 필요가 없어요. "빈자리"가 있는 곳만 우리 타겟이에요.

판정 기준:

상태등급
온라인 매거진·뉴스레터·카드뉴스 아무것도 없음A급 (공백)
SNS나 카드뉴스 하나 있지만 정기 매거진 없음B급 (부분)
이미 정기 매거진·레터 운영 중C급 (이미 함)

A급만 먼저 공략 → B급은 A 소진 후 → C급은 레퍼런스 생긴 후

👤
박낭만

A급이 진짜 타겟이네.

🤖
에이전트(영업팀)

정확해요. A급 = 우리 본진.

보험 때도 "만기 다가온 고객"이 제일 좋은 타겟이었잖아요 — 타이밍이 맞는 사람. 매거진 영업도 "아직 아무것도 없는 곳"이 지금 타이밍인 곳이에요.

3단계: 발행 유무 확인 방법
👤
박낭만

발행하는지 안 하는지 어떻게 알아?

🤖
에이전트(영업팀)

홈페이지 스크리닝으로 알아요:

  1. 홈페이지에 "소식/뉴스룸/웹진" 메뉴 있는지
  2. 뉴스레터 구독 버튼 있는지
  3. 카드뉴스·웹진 게시판이 정기 업데이트되는지
  4. 인스타·네이버포스트 정기 운영하는지

```
"기관명" 뉴스레터
"기관명" 웹진
"기관명" 매거진
```

이 검색에서 결과 안 나오면 → A급 가능성

👤
박낭만

직접 방문 없이 다 온라인으로 확인 가능해?

그 중 A급만 추려서 → 직접 컨택.

보험설계사 때 "잠재 고객 명단 만들기"랑 구조가 같아요. 스크리닝으로 거르고 → 좋은 타겟에 집중.

박낭만

보험할 때는 만기일이 핵심이었는데, 여기선 "다음 액션"이 핵심이네.

에이전트(영업팀)

정확해요. 보험은 만기라는 자연스러운 타이밍이 있어요. 매거진 영업은 타이밍을 내가 만들어야 해요.

"다음 액션 날짜"가 없으면 → 연락 미루다가 잊어버림.

매주 월요일 아침 → 이번 주 "다음 액션"이 된 곳 확인 → 그날 연락. 이게 리드DB 운영 루틴이에요.

리드DB 운영 규칙

> DB를 만들고 나서 어떻게 유지하는가

핵심 원칙 3가지

원칙 1: 매주 업데이트
  • 컨택 날짜
  • 반응 (응답/거절/검토중)
  • 다음 액션 내용 + 날짜
원칙 2: 다음 액션 날짜 항상 있게

"다음 액션 없음"은 → 영원히 방치 상태

원칙 3: 죽은 리드 정기 정리
  • 6개월 뒤로 다음 액션 미루기 (재시도)
  • C급으로 등급 강등
  • "장기 육성" 리스트로 이동 (뉴스레터만 발송)

정리하지 않으면 → DB가 죽은 리드로 가득 차서 실제 A급을 찾기 어려워짐

주간 루틴 (매주 월요일 아침 15분)

시간작업
5분"이번 주 다음 액션" 필터링 (날짜가 이번 주인 것)
5분지난 주 컨택 반응 업데이트
5분새 스크리닝 결과 추가 (3~5곳)

이 루틴만 지키면 → DB가 항상 살아있음

등급 변경 원칙

상황등급 변경
B급인데 새 예산 확보 소식B → A
A급인데 이미 다른 곳과 계약A → C
거절했던 C급인데 담당자 바뀜C → B (재접근 가능)
3번 무응답A → B 또는 C

등급 변경일도 기록 권장 (왜 바뀌었는지 메모)

단계별 액션 가이드

단계반응다음 액션
1차 컨택무응답1주 후 팔로업 (다른 채널)
2차 팔로업무응답2주 후 마지막 시도
3차 시도무응답6개월 후 재시도 or C급 이동
응답 옴관심샘플 발송 + 미팅 제안
미팅 완료검토 중1주 내 견적서·제안서 발송
거절거절이유 기록 후 6개월 뒤 재시도

도구 선택

규모추천 도구
리드 50개 이하구글 시트 (무료)
리드 50~200개구글 시트 + 필터 활용
리드 200개 이상Notion 데이터베이스 또는 HubSpot CRM 무료 버전

처음엔 구글 시트로 시작해서 — 리드가 많아지면 CRM 도구로 이전.

주의사항

  • 보도자료: 단발성, 사건 발생 시만 → A급 판정 가능
  • SNS 카드뉴스: 비정기, 산발적 → B급
  • 매달 정기 발행 → C급
  • 분기별 발행 → 정기성 있으면 C급 고려
