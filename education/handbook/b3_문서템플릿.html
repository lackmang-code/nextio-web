1단계: 왜 템플릿이 필요해?
👤
박낭만

에이전트야, 문서 템플릿을 왜 만들어야 해? 그냥 그때그때 새로 쓰면 안 돼?

🤖
에이전트(총무팀)

박낭만 씨가 20년간 대학 행정 했으면 — "표준 서식"의 중요성을 잘 알겠죠?

대학에서 품의서 양식이 정해져 있는 이유가 뭐예요? "그때그때 알아서 쓰세요"가 아니라 양식이 있는 이유?

👤
박낭만

누가 써도 같은 형식이 나와야 하고, 실수를 줄이기 위해서지.

🤖
에이전트(총무팀)

정확해요. 그게 기업 문서 템플릿의 이유랑 똑같아요.

사업에서 템플릿이 없으면 일어나는 일:

  1. 견적서를 보낼 때마다 형식이 달라져 → 고객이 전문성에 의문
  2. 부가세 빼먹거나 계산 실수 → 나중에 금액 분쟁
  3. 매번 처음부터 만들어야 해 → 시간 낭비

대학 행정처에 표준 서식 있는 것처럼 — 내 사업에도 표준 서식이 있어야 해요.

그리고 한 가지 더: 브랜드 일관성. 견적서·청구서·레터헤드가 전부 같은 로고·컬러·폰트를 쓰면 → 받아보는 사람이 "이 회사는 체계가 있다"고 느껴요.

🎓 강사 노트 ▼ 펼치기

비전문가에게 문서 템플릿의 중요성을 설명할 때 그들의 직업 경험을 레퍼런스로 쓰는 것이 핵심이다. 박낭만의 대학 행정 경험이 바로 그 레퍼런스다. 에이전트는 항상 상대의 언어로 설명해야 한다.

박낭만 씨 교육 행정 대행 서비스엔 이 중에서 어떤 게 가장 필요할 것 같아요?

👤
박낭만

견적서, 청구서, 납품 확인서는 당연히 필요하고. 수정 요청서는 뭔지 잘 모르겠어.

🤖
에이전트(총무팀)

수정 요청서가 사실 가장 중요한 서식 중 하나예요.

이런 상황이 생겨요:

  • 고객: "A 서비스 해달라고 계약했는데, 추가로 B도 해줄 수 있어요?"
  • 나: "그건 계약 범위 밖이에요"
  • 고객: "그냥 해줄 수 있지 않아요?"

이때 수정 요청서가 없으면 — 말로만 하다가 무료로 해주거나, 나중에 "왜 청구하냐"고 분쟁이 생겨요.

수정 요청서는: "추가 요청 내용 / 비용 / 일정 변경"을 문서로 남기는 것. 서명받으면 근거가 돼요.

대학 행정에서 "품의서 없이는 집행 못 한다"는 원칙처럼 — 수정 요청서 없이는 추가 작업 안 한다는 원칙이에요.

👤
박낭만

그거 중요하겠다. 대학에서도 구두 지시는 나중에 문제 생기더라고.

🤖
에이전트(총무팀)

맞아요. 구두로만 하면 나중에 "그런 말 한 적 없다"가 돼요. 특히 추가 비용이 생기는 상황에서 더 그래요.

혼동하기 쉬운데 완전히 달라요.

서식시점목적
견적서서비스 전"이 가격에 하겠습니다" 제안
거래명세서서비스 완료 후"이 서비스를 완료했습니다" 확인

문제가 생기는 경우: 견적서 없이 일 시작하면 → 금액 분쟁 / 청구서만 보내고 거래명세서 없으면 → "뭘 완료했냐"고 분쟁

세 가지가 다 있어야 분쟁을 막을 수 있어요.

👤

대학에서는 지출결의서에 이런 게 다 들어가 있었는데, 이제 내가 이걸 다 만들어야 하는구나.

🤖
에이전트(총무팀)

맞아요. 대학은 시스템이 다 갖춰져 있었는데, 1인 창업은 처음부터 만들어야 해요. 근데 한 번 만들어두면 — 그다음부터는 내용만 바꿔서 쓰면 돼요.

4단계: Excel 자동 계산 설정
👤
박낭만

견적서를 Excel로 만들면 어떻게 해? 수식을 어떻게 넣어?

🤖
에이전트(총무팀)

가장 기본적인 수식 3가지만 알면 돼요:

```
=C5*D5
```
(C5=수량, D5=단가라고 가정)

```
=SUM(E5:E10)
```
(E5~E10이 금액 열이라고 가정)

```
=E11*0.1
```
(E11이 합계라고 가정)

```
=E11+E12
```

이 네 줄 수식이 견적서의 전부예요. 새 견적서 만들 때 수량·단가만 바꾸면 금액·합계·부가세가 자동으로 계산돼요.

👤
박낭만

그거 몰랐어. 매번 계산기 두드려야 하는 줄 알았는데.

🤖
에이전트(총무팀)

Excel 수식 쓰면 계산 실수가 없어요. 그리고 한 번 만들어두면 영원히 써요.

추가 팁: 금액 셀 서식을 "₩ 통화 형식"으로 설정하면 — 숫자 입력만 해도 "₩ 1,000,000"처럼 자동으로 표시돼요.

핵심 수식 4개

① 금액 계산 (수량 × 단가)

E5셀에 입력:
```excel
=C5*D5
```
E6~E8까지 복사 (드래그 또는 Ctrl+D)

② 공급가액 합계

E9셀에 입력:
```excel
=SUM(E5:E8)
```

③ 부가세 (10%)

E10셀에 입력:
```excel
=E9*0.1
```

④ 최종 합계

E11셀에 입력:
```excel
=E9+E10
```

금액 셀 서식 설정

금액이 들어가는 열(E열) 전체 선택 → 우클릭 → 셀 서식:

  • 표시 형식: 통화 → ₩ #,##0 선택
  • 결과: 숫자 입력 시 자동으로 ₩ 1,000,000 형태 표시

유효기간 자동 날짜

견적서에 "발행일로부터 30일" 유효기간 자동 계산:

```excel
오늘 날짜: =TODAY()
유효기간: =TODAY()+30
```

날짜 셀 서식: YYYY년 MM월 DD일 형태

5단계: 브랜드 적용하는 방법
👤
박낭만

이 서식들에 회사 로고랑 색깔은 어떻게 넣어?

🤖
에이전트(총무팀)

Excel, Word 둘 다 로고 삽입이 돼요.

  1. 상단에 빈 행 3~4개 만들기
  2. 삽입 → 그림 → 로고 파일 선택
  3. 크기 조절 (비율 고정 잊지 말기)
  4. 회사명·연락처 텍스트 옆에 배치
  1. 머리글(Header) 영역에 로고 + 회사 정보
  2. 바닥글(Footer) 영역에 연락처·웹사이트
  3. 이 양식을 템플릿(.dotx)으로 저장 → 이후 새 문서를 열 때 자동 적용
  • 표 테두리 색상 → 브랜드 메인 컬러
  • 제목 행 배경 → 브랜드 배경 컬러
  • 강조 숫자(합계) → 브랜드 포인트 컬러

이렇게 하면 모든 서식이 같은 브랜드 느낌이에요.

👤
박낭만

Word 머리글에 로고 넣는 거 몰랐어. 그걸 하면 모든 페이지에 자동으로 들어가는 거야?

🤖
에이전트(총무팀)

맞아요. 머리글에 한 번 넣으면 — 문서가 몇 페이지가 되든 모든 페이지에 자동으로 들어가요.

이게 레터헤드 만드는 가장 쉬운 방법이에요.

🎓 강사 노트 ▼ 펼치기

Word 머리글/바닥글 기능은 행정 경험자에게 "아, 그게 그렇게 작동하는구나"라는 발견이 되는 경우가 많다. 에이전트가 이런 발견을 유도하는 것이 선배 역할이다.

레터헤드(공문 서식) 구성 가이드

> Word 머리글/바닥글 기능으로 제작 — 모든 페이지에 자동 적용

레터헤드 구조

```
┌─────────────────────────────────────────────────┐
│ [머리글 Header] │
│ [로고] [회사명] │
│ [사업자 등록번호] │
│ [주소] [전화] [이메일] │
├─────────────────────────────────────────────────┤
│ [본문 영역] │
│ │
│ 수신: │
│ 날짜: 문서번호: │
│ 제목: │
│ │
│ [본문 내용] │
│ │
│ │
│ [서명 영역] │
│ 위와 같이 알려드립니다. │
│ [회사명] [직함] [서명] │
├─────────────────────────────────────────────────┤
│ [바닥글 Footer] │
│ www.회사홈페이지.com | 이메일 | 전화 │
└─────────────────────────────────────────────────┘
```

Word 설정 방법

1. 머리글에 로고+회사 정보 넣기
  1. [삽입] → [머리글] → [머리글 편집]
  2. 왼쪽: 로고 이미지 삽입 (비율 고정)
  3. 오른쪽: 회사명 (굵게, 16pt) + 사업자번호·주소·연락처 (일반, 9pt)
  4. 머리글 영역 높이: 3~4cm
2. 바닥글에 연락처 넣기
  1. [삽입] → [바닥글] → [바닥글 편집]
  2. 가운데: 홈페이지 | 이메일 | 전화 (수평 구분선으로 분리)
  3. 얇은 선 위에 폰트 9pt
3. 브랜드 컬러 적용
  • 머리글 아래 얇은 선 → 브랜드 메인 컬러
  • 회사명 → 브랜드 메인 컬러
  • 바닥글 선 → 브랜드 메인 컬러
4. 템플릿(.dotx)으로 저장
  1. [파일] → [다른 이름으로 저장]
  2. 파일 형식: "Word 서식 파일 (*.dotx)"
  3. 파일명: 레터헤드공문서식템플릿.dotx
  4. 이후 새 문서 열 때: 이 파일을 열면 자동으로 레터헤드 적용

📁 영업/
