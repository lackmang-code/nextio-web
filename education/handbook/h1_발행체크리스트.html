1단계: 발행 = 공정이다
👤
박낭만

매거진 발행에 체크리스트가 필요해? 품질관리처럼?

🤖
에이전트(편집제작팀)

완전히 같아요. 발행 = 공정이에요.

제조업 공정매거진 발행
원자재 입고원고 마감 (콘텐츠 완결)
가공·조립빌드 (HTML 통합)
품질 검사QA (링크·반응형·폰트)
출하 승인발행 승인 (대표 OK)
출하배포 (업로드·URL 공개)
사후 관리발행 후 아카이브·마케팅 전달

ISO 공정에서 게이트 통과 못 하면 다음 단계 못 가잖아요.
매거진도 D-게이트 통과 못 하면 발행 못 해요.

👤
박낭만

D-게이트가 뭐야?

🤖
에이전트(편집제작팀)

통과하지 못하면 다음 단계로 못 넘어가는 강제 정지점이에요.

D-게이트기준통과 못 하면
① 콘텐츠 LOCK전체 원고 완결·팩트체크 완료빌드 착수 금지
② QA 통과링크·반응형·표시 오류 0대표 제출 금지
③ 대표 확인대표 발행 OK 수령외부 공개 금지

품질관리에서 "OK 없이 출하 금지" 원칙과 같아요.

🎓 강사 노트 ▼ 펼치기

품질관리 출신에게 D-게이트는 즉각 이해 가능. 제조업 공정 게이트 비유로 접근하면 체크리스트 전체 구조가 쉽게 잡힘.

2단계: 7개 단계 체크리스트 구조
👤
박낭만

체크리스트가 몇 개 단계야?

🤖
에이전트(편집제작팀)
단계이름역할
1콘텐츠 완결성원고 전부 완성·팩트체크
2발행본 빌드HTML 통합·표지·목차
3품질 검수 (QA)링크·반응형·표시 확인
4배포 자산OG 메타태그·썸네일
5호스팅·배포업로드·URL 확인
6발행 승인대표 최종 확인
7발행 후아카이브·마케팅 전달
  • 1단계 끝: D-게이트 ① (콘텐츠 LOCK)
  • 3단계 끝: D-게이트 ② (QA 통과)
  • 6단계 끝: D-게이트 ③ (대표 확인)
3단계: 콘텐츠 LOCK — 이 이후 수정 금지
👤
박낭만

콘텐츠 LOCK이 뭐야?

🤖
에이전트(편집제작팀)

D-게이트 ① 통과 후 — 본문 내용 변경 금지예요.

이유:

  • 빌드가 시작된 이후 본문이 바뀌면 → 빌드를 다시 해야 함
  • 팩트체크가 끝난 이후 수정하면 → 다시 팩트체크 필요

조립 시작 후에 설계 변경하면 — 이미 조립된 부품 다 해체해야 하잖아요.
콘텐츠 LOCK = "설계 확정, 이후 변경 금지"

  1. 전체 원고 최종고 완료
  2. 팩트체크 완료 (수치·인용·날짜 출처)
  3. 오탈자·맞춤법 교정 완료
  4. AI 제작 고지 삽입 ("본지는 Claude로 제작됨")
  5. 불확실 내용 → 대표에게 별도 보고 완료
4단계: QA — 기계 검사 vs 시각 검사 분리
👤
박낭만

QA를 어떻게 해? 품질검사처럼 기준이 있어야 하잖아.

🤖
에이전트(편집제작팀)

두 가지로 나눠요.

  • markitdown 텍스트 추출 → 내용 누락·깨짐 확인
  • 링크 클릭 테스트 → 깨진 링크 0
  • 이미지 경로 → 상대경로 확인
  • 브라우저에서 실제 렌더링 확인
  • 모바일 반응형 확인
  • 폰트·레이아웃 최종 판단

AI는 텍스트·링크 오류는 정확하게 잡지만
"이 레이아웃이 보기 좋은가"는 사람이 판단해야 해요.
품질관리에서 자동 검사와 시각 검사를 분리하는 것과 같아요.

5단계: OG 메타태그 — 링크 공유 미리보기
👤
박낭만

OG 메타태그가 뭐야?

🤖
에이전트(편집제작팀)

카카오톡, SNS에서 링크 공유할 때 나오는 미리보기 화면이에요.

OG 메타태그가 없으면:

  • 링크 공유 시 URL만 보임 → 클릭률 낮음

OG 메타태그가 있으면:

  • 링크 공유 시 이미지+제목+설명 보임 → 클릭률 높음
태그내용크기 기준
og:title호 제목60자 이내
og:description한 줄 소개120자 이내
og:image표지 썸네일1,200×630px
og:url발행 주소정확한 URL

이 크기가 LinkedIn, 카카오톡, Facebook 공통 표준이에요.
이 크기가 아니면 플랫폼마다 잘리거나 비율이 틀어져요.

  • 원고 완성 → 빌드 → QA → 대표 제출까지
작업담당
홍보·SNS 배포·구독자 모집마케팅팀

팀이 나 혼자라면 — 모든 역할을 시간 분리해서 한다.
"지금 나는 편집제작팀 모드 → 발행 완료 → 마케팅팀 모드로 전환"

7단계: 표준 체크리스트를 호별로 복사해 쓰는 법
👤
박낭만

매달 다 새로 만들어?

🤖
에이전트(편집제작팀)

아니요. 표준 양식을 복사해서 호별 파일로 쓰는 방식이에요.

ISO 공정과 같은 원리예요:

  • 표준 공정서 (변경 안 함)
  • 공정 이력서 (호별 복사, 날짜·결과 채움)
  1. 발행체크리스트표준.md = 원본 (매번 동일, 수정 금지)
  2. 새 호 시작 → 발행체크_vol01.md로 복사
  3. [ ] → [x]로 채우면서 진행
  4. 완료 후 하단 "발행 기록"에 날짜·URL 기록

팀 간 경계선 정리

> 발행 과정에서 어떤 팀이 무엇을 하는지 명확화

왜 경계선이 필요한가?

그래도 "지금 나는 어느 팀 모드인가"를 분리해야 한다.

역할이 섞이면:

  • 편집하다가 영업 생각 → 집중 분산
  • 마케팅 계획 세우다가 원고 수정 → 둘 다 부실

팀별 역할 분리표

편집제작팀콘텐츠·빌드·QA·대표 제출까지단계 1~6 (★ D-게이트 ③까지)
마케팅팀홍보·SNS·구독자 모집·보도자료 배포단계 7 이후 (발행 완료 알림 받은 후)
총무팀도메인·호스팅 계약·DNS 설정단계 5 지원
