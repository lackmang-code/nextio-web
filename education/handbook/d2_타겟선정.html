1단계: 타겟 선정이 뭐야?
👤
박낭만

에이전트야, 리드 유형은 정했어. 그런데 같은 유형 안에서도 누구한테 먼저 연락해야 해?

🤖
에이전트(영업팀)

바로 그게 "타겟 선정"이에요.

리드 유형 정의 = "어떤 종류의 사람/기업"
타겟 선정 = "그 중에서 지금 당장 연락할 구체적인 곳"

공무원 업무로 비유하면:

  • 리드 유형 = "소상공인 지원 대상"
  • 타겟 선정 = "이번 달 사업비 집행 가능한 점포 20곳 리스트"

선정 기준이 없으면 — 운에 맡기게 돼요.

👤
박낭만

선정 기준이 뭐야?

🤖
에이전트(영업팀)

핵심 기준 하나: "빈자리가 있는 곳"

빈자리 = "지금 내가 해결해줄 수 있는 문제가 해결되지 않고 남아있는 상태"

  • 디지털 전환 아직 안 한 식당 → 빈자리 있음
  • 이미 스마트오더·배달앱 다 됐음 → 빈자리 없음

빈자리 없는 곳에 아무리 좋은 제안을 해도 — 필요 없는 물건은 안 사요.

🎓 강사 노트 ▼ 펼치기

공무원 출신은 "대상 선정"이라는 개념에 익숙하다. 예산 집행 대상 기준을 정하는 것처럼 타겟 선정도 같은 원리다.

2단계: 빈자리 확인 방법
👤
박낭만

빈자리를 어떻게 알아?

🤖
에이전트(영업팀)

디지털 전환 컨설팅이라면 — 가장 빠른 방법:

  • 네이버 지도에 있는지 (리뷰·사진 있는지)
  • 배달앱(배민·쿠팡이츠)에 등록됐는지
  • 홈페이지/SNS 있는지

있으면 → 이미 어느 정도 함 (C등급 가능성)
없으면 → 빈자리 큼 (A등급 가능성)

  • 스마트 POS 사용하는지
  • QR 메뉴판 있는지
  • 예약 앱 연동됐는지
👤
박낭만

그러면 인터넷 검색으로 먼저 거르면 되겠네.

🤖
에이전트(영업팀)

맞아요. "사전 스크리닝"이라고 해요.

시간 아끼는 방법:

  1. 네이버·구글 지도 검색 → 리뷰·사진·웹사이트 있는지 확인
  2. 배달앱 검색 → 등록 여부
  3. 인스타·페이스북 계정 있는지

이 3가지 없으면 → A등급 가능성 높음 → 방문 또는 연락 우선

3단계: A/B/C 등급 분류
👤
박낭만

A/B/C 기준을 좀 더 구체적으로 알려줘.

🤖
에이전트(영업팀)

디지털 전환 컨설팅 기준으로 예시:

등급기준예시
A (즉시 공략)빈자리 있음 + 지금 당장 필요한 상황 + 결정권자 접근 가능식당 사장이 직접 운영, 배달앱 미등록, "요즘 손님이 줄었다"고 함
B (정기 팔로업)빈자리는 있는데 타이밍·예산이 불확실직원이 있어서 사장이 바빠 보임, 일부 디지털화는 됐음
C (제외 또는 방치)이미 다 됐거나, 필요 없다고 판단이미 스마트오더·배달·SNS 다 운영 중

A부터 공략, C는 건너뛰기.

👤
박낭만

C를 억지로 설득하면 안 돼?

🤖
에이전트(영업팀)

시간 낭비예요. "필요 없는 사람을 설득"하는 건 에너지가 10배 들어요.

그 시간에 A 10곳 더 연락하는 게 훨씬 효과적이에요.

공무원 때도 그랬잖아요 — 지원 의지 없는 사업자한테 계속 설득하다 보면 시간만 가는 거.

4단계: 함정 — "예산 큰 곳"이 항상 타겟이 아니다
👤
박낭만

지원사업비 받은 큰 가게를 타겟으로 하면 안 돼?

🤖
에이전트(영업팀)

조심해야 해요. 빠지기 쉬운 함정이에요.

"예산 큰 곳 = 우리 타겟"이 아닐 수 있어요.

왜냐면 예산이 크고 규모가 큰 가게는:

  • 이미 디지털 전환 다 됐을 수도 있음 (빈자리 없음)
  • 이미 전담 IT 업체 계약 중일 수 있음 (빈자리 없음)
  • 의사결정 느림 (여러 명 결재)

반면, 예산이 적어 보이는 소규모 식당이더라도:

  • 아직 아무것도 안 됐으면 → A등급
  • 사장이 바로 결정 → 계약 빠름

"크다"가 아니라 "빈자리 있고 결정권자 접근 가능한 곳"이 진짜 타겟이에요.

👤
박낭만

맞아, 공무원 때도 규모 큰 사업자가 지원받기 어려운 경우 많았어.

🤖
에이전트(영업팀)

그 경험이 지금 완전히 써먹히는 거예요. "지원 필요한 곳"과 "지원 받을 수 있는 곳"이 다르듯이 — "영업 필요한 곳"과 "계약될 수 있는 곳"이 달라요.

  • 배달앱 등록 아직 안 됨

  • 예약 시스템 없음

그리고 사장이 "뭘 해야 할지 막막하다"는 상황 → 에이전트(컨설턴트)가 필요한 순간.

어떻게 찾나:

  • 네이버 지도 "최근 오픈" 필터
  • 동네 SNS 그룹 (새로 생긴 가게 공지)
  • 직접 동네 걷다 보면 "GRAND OPEN" 현수막

박낭만

오, 신규 오픈 가게를 먼저 보면 되는구나.

막 열었을 때 → 디지털 세팅 필요 (A등급)

타겟 등급 기준 (A/B/C)

> 스크리닝 결과를 등급으로 분류 — 에너지 배분의 기준

A/B/C 등급 정의

A등급 — 즉시 공략
  • 빈자리가 명확히 있음
  • 지금 당장 필요한 상황 (신규 오픈, 최근 문제 발생)
  • 결정권자 접근 가능 (사장 직접 운영)
  • 예산 있음 (또는 확인 가능)
  • 막 오픈했는데 디지털 아무것도 없는 식당
  • 사장이 직접 "요즘 배달 주문이 없다"고 토로한 카페
  • RISE 사업단인데 정기 발행물 전혀 없음
B등급 — 정기 팔로업
  • 빈자리는 있는데 타이밍·예산 불확실
  • 관심은 있는 것 같지만 결정 안 함
  • 현재 다른 공급자와 검토 중
  • SNS 있는데 관리가 부실한 가게 (필요하긴 한데 아직 결정 못 함)
  • 담당자가 있는데 위에 승인받아야 한다는 곳
  • 글로컬대학 사업단 (예산 크지만 정기물 공백)
C등급 — 제외 또는 자동화 유지
  • 이미 비슷한 서비스 이용 중 (빈자리 없음)
  • 명확히 필요 없다고 한 곳
  • 예산 없음 + 개선 의지 없음
  • 이미 웹진·뉴스레터 정기 발행 중인 연구기관
  • 모든 디지털 전환 이미 완료된 대형 프랜차이즈

등급별 에너지 배분

등급전체 리드 중 비율쓰는 시간
A20%70%
B40%25%
C40%5%

대부분의 리드가 B나 C → 정상. A에 집중하는 것이 핵심.

등급 변화 추적

등급은 고정이 아닌 변동:

변화트리거
B → A새 예산 확보, 타이밍 도래, 담당자 교체
C → B이전 공급자와 계약 종료, 새 필요 발생
A → C이미 다른 곳과 계약 완료

리드DB에 "등급 변경일"을 기록하면 — 언제 다시 접근할지 판단 가능

타겟 스크리닝 방법

> 많은 후보 중 실제로 연락할 곳을 빠르게 걸러내는 방법

스크리닝이란

스크리닝 = "연락 전에 미리 거르기"

방문·전화 전에 온라인으로 먼저 확인 → 빈자리 있는 곳만 실제 컨택

온라인 스크리닝 3단계

1단계: 지도 앱 확인
  • 네이버 지도/구글 지도에 등록됐는지
  • 사진 있는지 (사장이 직접 올린 vs 방문자가 올린)
  • 최근 리뷰 있는지 (활성도 확인)
  • 홈페이지/SNS 링크 있는지
상태의미
지도에 없음디지털 존재감 제로 → 빈자리 큼
지도 있음 + 사진 없음아직 기본도 안 됨
지도+사진+SNS 링크이미 어느 정도 함
2단계: SNS 확인
  • 인스타그램 검색 (상호명 또는 주소)
  • 네이버 스마트플레이스 (리뷰·메뉴 업데이트 빈도)
  • 페이스북 페이지
상태의미
SNS 없음빈자리 있음
있는데 마지막 게시물이 6개월 전운영 포기 상태 → C
활성화됨이미 관리 중
3단계: 온라인 플랫폼 등록 확인 (업종별)
업종확인 플랫폼
식당/카페배달의민족, 쿠팡이츠 검색
쇼핑스마트스토어, 쿠팡 판매자
서비스업네이버 예약, 카카오예약
미용/헬스강남언니, 카카오뷰티

해당 플랫폼에 없으면 → 빈자리
